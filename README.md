# Tetris
1. 项目概述

1.1 项目简介
俄罗斯方块是一款具有全球影响力的经典益智类电子游戏，由苏联计算机科学家阿列克谢·帕基特诺夫于1984年发明。本程序在Windows平台上使用Visual Studio 2026作为开发工具，采用C++编程语言实现了俄罗斯方块游戏的核心功能，提供了完整的游戏体验。
本项目不仅是对经典游戏的复现，更是一个面向现代开发环境的游戏开发实践，展示了在Windows平台上使用C++进行控制台游戏开发的实践方法。

1.2 开发环境
操作系统: Windows
开发工具: Visual Studio 2026
编程语言: C++

1.3 项目特点
本游戏具有完整的俄罗斯方块游戏界面设计，包括游戏区域、分数显示和操作提示，支持7大类19种不同形状的俄罗斯方块，能够实现方块的旋转、移动、加速下落等基本操作，具有消行计分和游戏结束判断机制，是一个比较完善的俄罗斯方块游戏项目。

2. 系统设计架构

2.1 总体架构
程序采用模块化设计，主要包括以下模块：
1.界面显示模块：负责游戏界面的绘制和更新；
2.方块控制模块：处理方块的生成、移动、旋转等操作；
3.游戏逻辑模块：实现游戏规则、计分、消行等逻辑；
4.用户输入模块：处理键盘输入事件；
5.游戏状态模块：管理游戏状态和棋盘数据；

2.2 数据流设计
用户输入 → 输入处理 → 游戏逻辑 → 界面更新
               ↓
            方块控制 → 碰撞检测
               ↓
            状态更新 → 计分消行

3. 详细模块设计

3.1 全局数据结构

3.1.1 棋盘数据结构
将方块的局部形状坐标（x, y）转换为棋盘全局坐标（boardX = newX + x, boardY = newY + y）。

3.1.2 方块结构体
通过继承体系统一管理方块共性（如颜色、坐标），同时允许各子类自定义旋转模式
 
3.2 核心功能模块

3.2.1 界面显示模块
弃用system("cls")命令，通过控制台API精准控制光标和文本属性，使用缓冲区计算后批量提交显示。Renderer::drawGame() - 主渲染函数，Renderer::setChar() / setString() - 设置缓冲区，Renderer::renderBuffer() - 渲染到屏幕，Renderer::clearBuffer() - 清空缓冲区。
 
3.2.2 方块控制模块
1. 方块生成系统：BlockFactory::createRandomBlock() 工厂模式创建随机方块， Block::
getX()/Block::getY()获取方块当前位置，Block::setPos()设置方块位置（通过构造函数隐含）。
 
2. 方块旋转算法：每个方块内部维护旋转计数（rotation），通过取模运算（rotation % 2 或 rotation % 4）确定当前旋转状态，动态返回对应的形状矩阵。
 
3.2.3 游戏逻辑模块
碰撞检测系统消行计分算法：使用line_clear函数执行消行操作。将指定行以上的所有行下移一行。
 
3.2.4 用户输入模块
使用_getch()和_kbhit()函数检测并获取键盘输入，GameEngine::processInput() - 分发按键事件，转换为方向指令，其中支持的操作包括：上箭头(72)为旋转，下箭头(80)为加速下落，左箭头(75)为左移，右箭头(77)为右移。p键控制游戏暂停（32）；ESC键退出游戏（27）。实现非阻塞输入检测，确保游戏响应的即时性；将原始键盘编码转换为统一的游戏控制指令。
 
3.2.5 游戏主循环
在main函数中进入下落循环：用chrono::steady_clock::now()进行获取当前时间，chrono::duration_cast<chrono::milliseconds>()进行时间转换, 实现了三种不同频率的事件处理：最高优先级GameEngine::processInput()函数进行处理输入,GameEngine::update()函数则用于更新游戏状态,画面渲染renderElapsed每33ms一次；最后用this_thread::sleep_for()控制循环频率。
4. 界面设计

	游戏主区域（Playfield）：由"+"、"-"、"|"字符构成的矩形边框，颜色为白色（颜色值7）游戏区域：10列×20行的网格，内部显示：已固定的方块：用"[]"字符对表示，颜色根据方块类型不同（I-青色，J-蓝色，L-橙色，O-黄色，S-绿色，T-紫色，Z-红色）；空白位置：用"."字符表示，颜色为灰色（颜色值8）当前活动方块：正在下落的方块，同样用"[]"表示，颜色对应其类型。

	信息显示区域（Side Panel）：位置：信息显示区域下方，坐标范围 x: 25-40，y: 15-21固定显示的操作说明："CONTROLS:"标题，白色（7），(25, 15)；"Left/Right: Move"，白色（7），(25, 16)；"Up: Rotate"，白色（7），(25, 17)；"Down: Speed up"，白色（7），(25, 18)；"Space: Hard drop"，白色（7），(25, 19)；"P: Pause"，白色（7），(25, 20)；"ESC: Quit"，白色（7），(25, 21)。

	操作提示区域：暂停状态：半透明黑色覆盖层（颜色0）；"PAUSED"标题，黄色（14）；"Press P to continue"提示，白色（7）。游戏结束状态：半透明黑色覆盖层（颜色0）；"GAME OVER!"标题，红色（12）；显示最终分数，白色（7）；"Press ENTER to restart"和"Press ESC to quit"选项，白色（7）。

5. 性能优化设计

5.1 内存优化
提高cpu利用率
 
5.2 渲染优化
只更新变化的区域，避免全屏刷新。使用双缓冲思想，先擦除旧位置，再绘制新位置。

5.3 输入处理优化
使用_kbhit()非阻塞检测键盘输入。在游戏循环中及时处理输入，避免输入延迟。

6. 错误处理与调试

6.1 编译警告处理
中文乱码问题，全改成英文就解决了

6.2 运行时错误预防
数组越界检查：在访问棋盘数组前检查坐标范围。
输入验证：验证用户输入的合法性。
状态一致性：确保游戏状态的正确转换。

7. 扩展性和维护性
每个功能模块相对独立，模块间通过清晰定义的接口通信，易于单独测试和修改。

8. 测试方案

8.1 单元测试
方块旋转测试：验证所有方块类型的旋转是否正确。
碰撞检测测试：测试边界碰撞和方块间碰撞。
消行计分测试：验证消行和计分逻辑。

8.2 集成测试
游戏流程测试：完整运行游戏，验证各模块协同工作。
用户交互测试：测试所有键盘操作的正确响应。
异常处理测试：测试游戏异常情况下的行为。

总结

本俄罗斯方块游戏程序设计充分考虑了游戏的完整性、可玩性和代码的可维护性。通过模块化设计和清晰的算法实现，确保了程序的稳定运行。程序具有良好的扩展性，可以根据需要增加更多功能。界面设计简洁明了，操作响应及时，为用户提供了良好的游戏体验。
程序在实现经典俄罗斯方块游戏核心功能的基础上，还加入了触底缓冲、消行计分等增强功能，提高了游戏的可玩性。
